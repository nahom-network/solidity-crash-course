# Applied Solidity

Welcome to the **Applied Solidity** module! This section of the Solidity Crash Course focuses on practical applications and advanced patterns in smart contract development.

Here, you will learn how to apply core concepts to build robust, structured, and realistic decentralized applications.

## Module Contents

This folder contains the following sub-topics, each featuring hands-on practice:

### 1. Inheritance
Learn how to use object-oriented inheritance in Solidity to create reusable and organized code structures.
* **Concepts Covered:** Base and derived contracts, overriding functions, code reuse strategies, and understanding the tradeoffs between inheritance and using libraries.
* **Directory:** `Inheritance/Practice Inheritance`

### 2. Voting
Build a functional voting system to understand governance and collective decision-making on the blockchain.
* **Concepts Covered:** Creating proposals, tracking member votes, executing proposals via calldata, and exploring the foundations of the Governor standard for protocol upgrades and management.
* **Directory:** `Voting/Build a Voting Contract`

## Getting Started

Navigate to any of the subdirectories and read their respective `README.md` files to begin. Each practice folder contains a Foundry project setup with `contracts` to modify and `tests` to verify your implementation.
