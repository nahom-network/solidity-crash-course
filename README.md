# Solidity Learning Repository

A comprehensive collection of Solidity smart contract practice exercises and projects, extracted from Alchemy's blockchain development course. This repository contains 11 modules covering fundamental to advanced Solidity concepts.

## Overview

This repository is organized into **4 main learning tracks**:

### 1. **Solidity Introduction** 📚

#### Resources

[Presentations](https://github.com/alchemyplatform/learn-solidity-presentations)
[Marp Tool](https://marp.app/)
[Foundry](https://book.getfoundry.sh/)

Get started with Solidity fundamentals:

- **Practice Data Types** - Work with bools, integers, and enums
- **Practice Solidity Functions** - Learn how to write and implement Solidity functions

### 2. **Address Interactions** 🔗

Learn how to interact with addresses and other contracts:

- **Practice Sending Ether** - Master the `.call` pattern for sending ETH
- **Practice Reverting Transactions** - Use `require` to handle transaction validation
- **Practice Calldata** - Send calldata between contracts
- **Build an Escrow** - Build a complete smart contract escrow system

### 3. **Reference Types** 📦

Work with Solidity's complex data structures:

- **Practice Arrays** - Master array operations and management
- **Practice Structs** - Create and manipulate custom data structures
- **Practice Mappings** - Work with key-value pair storage

### 4. **Applied Solidity** 🚀

Apply your knowledge to real-world scenarios:

- **Build a Voting Contract** - Create a smart contract voting system
- **Practice Inheritance** - Work with contract inheritance and polymorphism

## Repository Structure

Each module typically contains:

```
Module Name/
├── README.md          # Module-specific instructions
├── contracts/         # Solidity smart contracts
│   └── *.sol
└── tests/            # Foundry test files
    └── *.t.sol
```

## Getting Started

1. **Navigate to a module** - Choose a learning track and explore the corresponding folder
2. **Read the README** - Each module has a dedicated README with instructions
3. **Review contracts** - Examine the contract files in the `contracts/` directory
4. **Study the tests** - Check test files to understand expected behavior
5. **Practice** - Complete the exercises as outlined in each module
